소개

한 줄 설명

BaseMark는 사진을 “이해”하려는 앱이 아니라, 기준점을 잡아 비교 후보를 계산하는 앱입니다.

먼저 현장에서 잘 안 바뀌는 구조 기준점 두 개를 잡습니다. 그 기준으로 예상 위치를 계산한 뒤, 빠짐, 추가, 위치 차이를 후보로 보여주고 사람이 최종 확인합니다.

기준점 0 비교의 기준이 되는 구조점
체크포인트 0 도면 기준으로 예상되는 위치
관측 요소 0 사진에서 실제로 보이는 대상
후보 0 사람이 확인할 결과

기준점이란

문틀 끝, 창호 바깥 프레임 모서리, 벽 코너처럼 잘 안 바뀌는 구조점입니다. 아무 점이 아니라 “역할이 있는 점”이어야 합니다.

비율이란

절대 픽셀이 아니라 기준점 두 개 사이의 가로 비율과 세로 오프셋으로 위치를 표현합니다. 그래서 사진 크기와 각도가 달라도 비교 기준을 유지할 수 있습니다.

결과란

missing, extra, position_diff 세 종류만 보여줍니다. BaseMark는 자동 확정이 아니라 검토 후보 생성까지 맡습니다.

현장 비교

사진 한 장에서 기준점을 잡고, 보이는 대상을 찍고, 후보를 찾습니다.

이 화면만 보면 됩니다. 나머지 세부 JSON과 폼은 아래 고급 편집기로 접어 두었습니다.

1. 기준점 잡기
2. 보이는 대상 찍기
3. 후보 확인
기준점 역할 선택

먼저 기준점 역할을 고르세요. 예를 들어 창호를 기준으로 보면, 창호 바깥 프레임의 모서리를 찍어야 합니다.

먼저 기준점 두 개를 잡으세요.

같은 구조선 위의 두 점을 잡아야 합니다. 예를 들어 창호를 기준으로 본다면 “창호 바깥 프레임 좌상단 + 우상단”처럼 같은 프레임 기준으로 짝을 맞추세요.

어디를 클릭하나

움직이는 창짝이 아니라, 고정된 바깥 프레임 모서리나 벽 코너를 클릭합니다.

어떻게 짝을 맞추나

좌상단과 우상단처럼 같은 선, 같은 구조에서 두 점을 잡아야 비율이 의미를 가집니다.
