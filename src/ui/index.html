Fixed Identity

BaseMark is a comparison engine first.

Structural anchor selection, normalized coordinates, local alignment, and missing/extra/position_diff candidate generation are the product core. Storage and reports are the operating layer.

Pipeline

  1. Select stable anchors for one bounded local segment.
  2. Define normalized checkpoints against those anchors.
  3. Align the local field segment and project expected positions.
  4. Generate candidate evidence for human review.

Visual UI Focus

The engine screen pairs editable JSON with a canvas view. You can scan anchors, checkpoints, observed elements, and candidates without reading raw payloads first.

Canvas overlay Saved scenarios Human review cards

Reading Order

  1. Confirm scenario metadata and metrics first.
  2. Use the canvas to validate structure and point placement.
  3. Check summary cards and projected checkpoints.
  4. Use candidate cards before falling back to raw JSON.

Scenario Builder / 시나리오 빌더

카드형 폼으로 구간, 앵커, 체크포인트, 관측 요소를 먼저 정리하고 필요할 때만 JSON을 직접 수정합니다. Fill the cards first, then touch raw JSON only when needed.

구간은 한 번에 비교할 국소 장면 범위입니다. 앵커는 비교 프레임을 고정하는 구조 기준점입니다. 체크포인트는 상대 비율 좌표로 유지됩니다.

구간 정보 / Segment

현재 비교할 벽면 또는 국소 구간의 식별자와 라벨입니다.

앵커 / Anchors

문틀, 코너, 프레임처럼 비교 기준으로 삼을 구조점을 관리합니다.

체크포인트 / Checkpoints

정규화 좌표로 정의되는 예상 위치입니다. 엔진은 이 지점을 투영해 후보를 계산합니다.

관측 요소 / Observed Elements

현장 장면에서 실제로 보이는 요소입니다. 캔버스에서 직접 끌어 위치를 수정할 수 있습니다.

보조 JSON 편집기 / Raw JSON

사진 + 도면 사용 방식 / Photo + Drawing Workflow

BaseMark는 사진을 이해하는 범용 AI가 아니라, 도면 기준 구조를 이용해 사진의 국소 장면을 계산적으로 비교합니다.

  1. 도면은 구조 기준입니다. 문틀, 코너, 벽 경계 같은 앵커와 체크포인트의 상대 위치를 먼저 정의합니다.
  2. 사진은 현장 증거입니다. 같은 국소 구간에서 실제로 보이는 앵커와 관측 요소 좌표를 입력합니다.
  3. 엔진은 도면의 상대 좌표를 현장 사진 좌표로 투영해서 missing, extra, position_diff 후보를 계산합니다.
  4. 즉, 도면은 기준 프레임이고 사진은 비교 대상입니다. 최종 판정은 사람이 후보 카드와 캔버스를 보고 확정합니다.
