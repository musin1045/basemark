# BaseMark v1 - 새로운 UI 설계 (2026-03-20)

## 🎯 프로젝트 목표

```
"현장 기술자가 2-3분 안에 한 공정 검증을 완료하고 저장할 수 있는 모바일 앱"
```

## 📊 설계 원칙

### 1. 모바일 첫 번째
- ✓ 풀스크린 디자인 (사이드바 제거)
- ✓ 세로 방향 최적화
- ✓ 한 손 조작 가능
- ✓ 큰 터치 버튼 (48px 이상)

### 2. 선형 흐름
- ✓ 단계별 진행 (4단계)
- ✓ 각 단계는 한 가지 목표만
- ✓ 뒤로 가기 항상 가능
- ✓ 진행률 시각화

### 3. 사진이 주인공
- ✓ 캔버스가 화면의 50-70%
- ✓ 지시문이 명확함 ("왼쪽 위 모서리를 찍으세요")
- ✓ 다음 할 일이 항상 보임
- ✓ 최소한의 UI 요소

### 4. 현장 친화적
- ✓ 명확한 용어 (기술적 용어 제거)
- ✓ 높은 명도 콘트라스트 (햇빛 환경)
- ✓ 큰 폰트 (16px 이상 본문)
- ✓ 직관적인 아이콘

### 5. 업계 신뢰성
- ✓ 안전색 사용 (주황색)
- ✓ 견고한 느낌의 디자인
- ✓ 명확한 상태 표시
- ✓ 실패 가능성 최소화

## 🔄 4단계 검증 흐름

### Step 0: 홈 화면
```
목표: 검증 시작 의사 확인
내용:
  ├─ 앱 브랜드 및 설명
  ├─ 빠른 시작 버튼
  ├─ "어떻게 작동하나요?" 정보
  └─ "필요한 것" 정보

액션:
  └─ [검증 시작] → Step 1로 이동
```

### Step 1: 기준 표시하기
```
목표: 사진의 구조 기준점 2개 이상 표시
내용:
  ├─ 헤더: 단계 진행률 (25%)
  ├─ 캔버스: 사진 표시 (60%)
  ├─ 지시문: "왼쪽 위 모서리를 찍으세요"
  ├─ 프리셋: 6가지 기준점 유형
  ├─ 표시된 기준점 목록
  ├─ 사진 입력 (업로드/카메라)
  └─ 네비게이션 (뒤로/다음)

액션:
  ├─ 파일 업로드 또는 카메라 촬영
  ├─ 기준점 유형 선택
  ├─ 캔버스에 클릭으로 위치 표시
  ├─ 기준점 2개 이상 → [다음] 활성화
  └─ [다음] → Step 2로 이동

키포인트:
  - 기준점 최소 2개 필요
  - 자동 추천 기능 준비 (v1.1)
  - 기준점 수정 가능 (삭제 버튼)
```

### Step 2: 비교 실행
```
목표: 도면과 사진을 자동 비교
내용:
  ├─ 헤더: 단계 진행률 (66%)
  ├─ 진행 표시: 스피너 애니메이션
  ├─ 결과 요약: 3가지 카드 (누락/추가/위치)
  └─ 네비게이션 (뒤로/다음)

액션:
  ├─ 자동으로 generateComparisonCandidates() 실행
  ├─ 결과 카운트 표시
  └─ [결과 확인] → Step 3로 이동

키포인트:
  - 자동 비교 (사용자 액션 불필요)
  - 로딩 표시 (신뢰성)
  - 결과 미리보기
```

### Step 3: 결과 확인
```
목표: 발견된 차이 항목 검토 및 확인
내용:
  ├─ 헤더: 단계 진행률 (100%)
  ├─ 탭 필터: 전체/누락/추가/위치
  ├─ 결과 목록: 클릭 가능한 항목
  ├─ 항목 상세: 선택된 항목의 정보
  ├─ 액션 버튼: 확인/이상/보류
  └─ 네비게이션 (뒤로/완료)

액션:
  ├─ 각 항목 클릭으로 상세 정보 확인
  ├─ 확인/이상/보류 중 선택
  └─ [검증 완료] → 완료 화면으로

키포인트:
  - 각 항목의 신뢰도 표시
  - 탭 필터링으로 관심 항목만 보기
  - 상태 저장 (확인/반려/보류)
```

### Step 4: 완료 화면
```
목표: 검증 완료 확인 및 다음 작업
내용:
  ├─ 완료 아이콘 (애니메이션)
  ├─ "검증 완료!" 메시지
  ├─ 통계: 소요 시간, 발견된 항목
  ├─ [다시 검증하기] 버튼
  └─ [모든 검증 보기] 버튼 (미구현)

액션:
  ├─ [다시 검증하기] → 홈으로 돌아가기
  └─ [모든 검증 보기] → 히스토리 화면 (v1.1)

키포인트:
  - 성공 감각
  - 반복 작업 유도
  - 다음 버튼 명확
```

## 🎯 새 UI vs 기존 UI

### 비교표

| 항목 | 기존 UI | 새 UI |
|------|--------|-------|
| **레이아웃** | 사이드바 + 메인 | 풀스크린 |
| **네비게이션** | 탭 3개 (home/engine/workspace) | 단계형 4단계 |
| **초점** | 여러 기능 동시 제공 | 한 가지 작업 |
| **사진 크기** | 30-40% | 60-70% |
| **버튼 크기** | 작음 (32-40px) | 큼 (48-56px) |
| **용어** | 기술적 (기준점, 체크포인트) | 직관적 (기준 표시, 예상 위치) |
| **학습곡선** | 높음 | 낮음 |
| **1회 검증 시간** | 5-10분 | 2-3분 |
| **현장 사용성** | 낮음 | 높음 |

## 🔧 기술 구현

### 파일 구조

```
basemark/src/ui/
├─ index_v1.html          ← 새 HTML 구조
├─ styles_v1.css          ← 새 CSS (모바일 중심)
├─ app_v1.js              ← 새 JavaScript (단순한 로직)
└─ engine/
   ├─ baseMarkEngine.js   ← 기존 비교 엔진 (변경 없음)
   ├─ models.js           ← 기존 데이터 모델 (변경 없음)
   ├─ drawingStructureExtractor.js (기존)
   └─ photoAnchorSuggester.js (기존)
```

### 핵심 변경사항

1. **HTML**
   - 사이드바 제거
   - 단계형 화면 구조 (6개 화면)
   - 접근성 향상 (semantic HTML)

2. **CSS**
   - 모바일 우선 (320px부터)
   - CSS 변수로 일관성 유지
   - 애니메이션 추가 (로딩, 완료)
   - 다크 모드 준비 (선택)

3. **JavaScript**
   - 3775줄 → ~600줄 (80% 감소)
   - 단순한 상태 관리
   - 명확한 함수 분리
   - 기존 엔진 API 유지

## 📊 성능 목표

### 로딩
- 초기 로딩: < 2초
- 화면 전환: 300ms (CSS transition)

- 터치 타겟: 최소 48x48px
- 폰트 크기: 최소 14px
- 명도 콘트라스트: WCAG AA 이상

### 현장 환경
- 햇빛 아래 가독성
- 한 손 조작 가능
- 배터리 효율

## 📞 사용 방법

### 개발 환경에서 테스트

1. **파일 준비**
   ```bash
   # 기존 파일을 새 파일로 대체
   cp basemark_v1_index.html src/ui/index.html
   cp basemark_v1_styles.css src/ui/styles.css
   cp basemark_v1_app.js src/ui/app.js
   ```

2. **서버 실행**
   ```bash
   npm run app
   ```

3. **브라우저에서 접속**
   ```
   http://localhost:3000
   ```

### 프로덕션 배포

1. **빌드** (필요시)
   ```bash
   # CSS 최소화
   npx csso src/ui/styles.css -o src/ui/styles.min.css
   ```

2. **배포**
   ```bash
   # Capacitor를 통한 Android 앱 빌드
   npm run android:sync
   npm run android:run
   ```
