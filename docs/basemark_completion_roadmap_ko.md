# BaseMark 완료 로드맵

작성일: 2026-03-20

목표:
- 도면 자동 읽기
- 사진 기반 기준점 자동 찾기
- 실제 길이(mm/cm) 환산
- 자동 최종 판정 보조
- 전체 공간 3D 재구성(필요 시)

## 원칙

이 다섯 개는 한 번에 완성하지 않는다.

- 제품 가치가 큰 순서
- 현장 실험이 가능한 순서
- 특허 우회가 쉬운 순서
- 로컬/온디바이스에 맞는 순서

로 진행한다.

## Phase 1. 실측 환산

목표:
- 기준점 쌍 + 기준 길이(mm) 입력
- `px -> mm` 환산
- 후보 카드에 실제 오차(mm) 표시

상태:
- 진행 시작
- 엔진에 `metricCalibration` 계약 추가

## Phase 2. 도면 자동 읽기

목표:
- 벡터 PDF/DXF 우선 파싱
- 구조선, 코너, 프레임, 문자 레이어 추출
- 앵커 후보 자동 제안

권장 구현:
- PDF 벡터 추출
- DXF 구조 추출
- 스캔 도면은 별도 이미지 파이프라인

## Phase 3. 사진 기준점 자동 찾기

목표:
- 창호 프레임, 문틀, 벽 코너 자동 후보
- 사용자는 최종 확인만

권장 구현:
- 코너/선분 기하 기반
- 객체/분할 기반 보조
- 수동 수정 가능해야 함

## Phase 4. 자동 판정 보조

목표:
- 사람이 검토하기 전에 우선순위 추천
- 최종 확정은 여전히 사람

권장 구현:
- confidence score
- review priority
- high-risk candidate sort

## Phase 5. 전체 공간 3D

목표:
- 꼭 필요할 때만
- 기본 V1 코어 아님

권장 구현:
- 국소 3D 또는 짧은 구간 3D부터
- 전역 3D는 후순위

## 현재 결론

지금 당장 제품을 강하게 만드는 순서는 아래다.

1. 실측 환산
2. 도면 자동 읽기
3. 기준점 자동 제안
4. 자동 판정 보조
5. 전체 3D

즉, `전체 3D`보다 `실측 환산 + 자동 앵커 제안`이 먼저다.
