# BaseMark — 2026-03-21 정리

## 1. 현재 제품 정의
BaseMark는 기준세대와 도면을 기준으로, 현장의 정합성을 빠르게 확인하고 수정·보완하는 로컬 우선 현장 도구로 정의한다.

핵심은 다음과 같다.
- 기준세대 / 도면 대비 확인
- 현장 차이 후보 제안
- 사람 최종 확정
- 저장 / 보고서
- 로컬 우선 사용

## 2. 방향 재정의
BaseMark는 단순히 가볍기만 한 앱이 아니라, 휴대폰 성능 강화와 폰 내장 AI를 적극 활용해 현장 확인과 입력 부담 감소를 크게 끌어올리는 방향으로 간다.

단, 다음 원칙은 유지한다.
- AI는 보조만 담당
- 최종 판정은 사람
- 핵심 루프는 AI 없이도 돌아가야 함

## 3. 장기 진화 방향
BaseMark의 시작점은 비교·정합성 확인 도구이지만, 최종 진화 방향은 공사 현장 앱이다.

다만 핵심 차별점은 끝까지 비교·정합성 확인 기능에 둔다.
- 시작: 비교 / 정합성 확인
- 중간: 일정 / 공수 / 현장 기록 포함
- 최종: 비교 검증을 핵심 무기로 가진 공사 현장 앱

## 4. 무료 / 유료 전략
일정과 개인 공수는 무료 기능으로 크게 열어 배포를 늘리는 입구 기능으로 둔다.

### 무료
- 개인 일정
- 개인 공수
- 1인 비교 작업
- 로컬 저장
- 수동 백업
- 기본 보고서

### 유료
- 여러 명 공수 관리
- 팀 단위 조회
- 서버 백업
- 자동 동기화
- 팀 리뷰 / 통합 리포트
- 관리자 기능
- 고급 보조 기능

중요 원칙:
- 일정 / 공수는 무료 배포용 입구가 될 수 있음
- 하지만 앱의 본체 자리는 비교 작업이 유지해야 함

## 5. 제품 구조 원칙
권장 구조는 다음과 같다.
- 홈
- 비교 작업
- 리뷰 대기
- 일정
- 내 공수
- 보고서
- 팀 관리(PRO)
- 백업 / 동기화(PRO)

즉, 일정 / 공수는 전면 입구가 될 수 있지만, 제품 중심은 계속 비교 작업으로 유지한다.

## 6. 사용자 표시 용어
내부 엔진 용어 missing / extra / position_diff 는 사용자 화면에 그대로 노출하지 않는다.

### 카메라 오버레이
- 미시공
- 기준외
- 위치오차

### 검토 화면
- 미시공 후보
- 기준 외 시공 후보
- 시공 위치 오차 후보

### 보고서
- 미시공
- 기준 외 시공
- 시공 위치 오차

원칙:
- 실시간 화면은 짧게
- 검토는 조심스럽게
- 보고서는 정식 용어로

## 7. 도면 종류별 오버레이 프로필
오버레이는 하나로 고정하지 않고, 도면 종류에 따라 다르게 바뀌어야 한다.

예시:
- 평면도: 구조선 / 문 / 창 / 벽 중심
- 전기도: 스위치 / 콘센트 / 조명 포인트 중심
- 배관도: 경로 / 분기 / 연결점 중심
- 천장도: 격자 / 정렬 / 간격 중심

즉, 도면 종류별 오버레이 프로필을 적용해 현장 확인 기준을 자동 전환하는 구조로 간다.

## 8. 오버레이 겹치기 기능
오버레이는 단일 표시가 아니라, 여러 기준층을 겹쳐 보는 구조가 필요하다.

기본 레이어:
- 카메라 실사
- 기준 도면 / 기준세대
- 이상 후보 표시
- 메모 / 수정 지시 / 체크 상태

필수 제어:
- 레이어 켜기 / 끄기
- 투명도 조절
- 미세 정렬 보정
- 프리셋 모드

## 9. 도면 겹치기 규칙
도면은 기본적으로 1개를 중심으로 보고, 필요 시 최대 5개까지 겹쳐 볼 수 있게 한다.

원칙:
- 기본: 도면 1개
- 확장: 최대 5개
- 구조: 주도면 1개 + 보조도면 최대 4개
- 실사용 중심: 2~3개
- 5개 겹치기: 고급 검토 모드

## 10. 현재 기준 핵심 한 줄
BaseMark는 기준세대와 도면을 기준으로, 휴대폰 성능과 폰 내장 AI를 적극 활용해 현장의 정합성을 빠르게 확인하고 수정·보완하는 로컬 우선 현장 도구다.
