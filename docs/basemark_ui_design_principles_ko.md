# BaseMark UI 디자인 원칙

## 1. 제품 성격

BaseMark는 일반 소비형 앱이 아니라 현장 판단 도구다.  
따라서 디자인의 목표는 "보기 좋음"보다 다음 4가지에 있다.

- 지금 해야 할 일이 바로 읽힐 것
- 사진과 오버레이가 화면의 중심일 것
- 기준점, 예상 위치, 관측 요소, 후보가 즉시 구분될 것
- 모바일 현장에서 손가락으로 바로 조작될 것

## 2. 기본 방향

### 2-1. 인스타그램에서 가져올 것
- 큰 미디어 영역
- 카드형 정보 구조
- 여백이 넓은 단순한 레이아웃
- 한 화면 한 목적 중심

### 2-2. 인스타그램에서 가져오지 않을 것
- 장식 위주의 비주얼
- 과한 그라데이션
- 피드 소비형 인터랙션
- 정보보다 스타일이 앞서는 구성

## 3. 화면 위계

### 3-1. 첫 번째 위계: 현재 모드
- 설정
- 검사
- 검토

사용자는 항상 "내가 지금 어느 단계에 있는가"를 먼저 알아야 한다.

### 3-2. 두 번째 위계: 사진
- 메인 화면에서 가장 큰 영역은 항상 사진 또는 카메라다.
- 모든 오버레이는 사진을 방해하지 않고 보조해야 한다.

### 3-3. 세 번째 위계: 다음 행동
- 지금 해야 할 일 카드는 항상 짧은 문장 하나로 유지한다.
- 예:
  - 첫 번째 기준점을 찍으세요.
  - 두 번째 기준점을 찍으세요.
  - 보이는 대상을 찍으세요.
  - 후보 찾기를 누르세요.

### 3-4. 네 번째 위계: 결과
- 결과는 오른쪽 또는 하단 카드에 모은다.
- 캔버스를 누르면 후보가 강조되고, 후보 카드를 누르면 캔버스도 같은 위치를 강조한다.

## 4. 용어 원칙

### 기준점
- `앵커`보다 `기준점`을 기본 용어로 쓴다.
- 필요한 경우만 `기준점(앵커)`처럼 병기한다.

### 체크포인트
- `예상 위치`를 먼저 쓰고, 필요 시 체크포인트를 병기한다.

### 관측 요소
- `사진에서 실제로 보이는 대상`이라는 설명을 함께 둔다.

### 후보
- BaseMark는 자동 확정이 아니라 후보 제안이라는 점을 계속 유지한다.

## 5. 색상 원칙

- 기준점: 검정 또는 아주 짙은 남색
- 예상 위치: 파랑
- 관측 요소: 브라운 또는 올리브
- missing: 붉은 계열
- position_diff: 파랑 계열 강조
- extra: 오렌지 또는 황갈색

색은 많을수록 좋은 게 아니라, 역할이 즉시 읽히는 정도로만 제한한다.

## 6. 컴포넌트 원칙

### 사진 카드
- 가장 큰 카드
- 오버레이와 상태 캡션 포함

### 단계 카드
- 설정 / 검사 / 검토 3개만
- 한 줄 요약과 짧은 설명만 유지

### 상태 카드
- 기준점 수
- 관측 요소 수
- 후보 수

### 후보 카드
- 후보 타입
- 체크포인트
- 이유 코드
- 검토 버튼

## 7. 숨겨야 할 것

다음 항목은 기본 화면에서 숨기고, 고급 편집기로 내린다.

- 원본 JSON
- 세부 시나리오 텍스트
- 기술자용 진단성 입력
- 자주 쓰지 않는 편집 필드

## 8. 모바일 원칙

- 한 번에 한 가지 액션만 강조
- 버튼은 크게
- 세로 흐름 우선
- 스크롤 깊이 최소화
- 탭 대상은 넉넉하게

## 9. BaseMark에 맞는 디자인 한 줄 정의

BaseMark의 디자인은 "예쁜 현장 앱"이 아니라  
"사진 중심, 단계 중심, 후보 검토 중심의 현장 비교 도구"여야 한다.
