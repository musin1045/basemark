# BaseMark 도면 자동 읽기 계약

작성일: 2026-03-20

## 목적

도면 자동 읽기의 첫 번째 목표는 "도면 전체를 이해하는 것"이 아니다.

첫 번째 목표는 아래 두 가지다.

1. 벡터 도면에서 구조선을 읽는다.
2. 기준점 후보를 자동으로 제안한다.

즉, BaseMark에서 도면 자동 읽기는 `구조 앵커 후보 생성기`로 시작한다.

## V1 입력

현재 첫 슬라이스는 아래 입력만 지원한다.

- SVG 벡터 도면 텍스트

이유:
- 벡터 구조가 명확하다.
- 선분, 사각 프레임, 폴리라인을 직접 읽기 쉽다.
- PDF/DXF로 가기 전 가장 작은 테스트 조각이다.

## V1 출력

현재 출력은 아래 세 가지다.

1. 선분 목록
2. 코너 목록
3. 앵커 후보 목록

앵커 후보 종류:
- `frame_left_top`
- `frame_right_top`
- `frame_left_bottom`
- `frame_right_bottom`
- `drawing_corner`

## V1 처리 범위

현재 추출기는 아래 요소를 읽는다.

- `<line>`
- `<rect>`
- `<polyline>`

처리 결과:
- 선분 생성
- 선분 끝점 클러스터링
- 사각 프레임의 4개 코너를 구조 기준점 후보로 제안

## 하지 않는 것

현재는 아직 아래를 하지 않는다.

- PDF 파싱
- DXF 파싱
- 스캔 도면 OCR
- 문/창 자동 분류
- 공간 이름 자동 이해
- 체크포인트 자동 생성

## 다음 단계

1. PDF 벡터 추출 추가
2. DXF 구조 추출 추가
3. 사각 프레임을 창호/문틀 후보로 분류
4. 도면 코너와 현장 기준점 자동 매칭 보조

## BaseMark 원칙

도면 자동 읽기는 BaseMark의 본체를 대체하지 않는다.

도면 자동 읽기는:
- 구조 앵커 후보를 제안하고
- 사람이 확인하기 쉽게 만들고
- 이후 정규화 좌표/국소 정합/후보 생성으로 이어지게 하는 보조층이다.
