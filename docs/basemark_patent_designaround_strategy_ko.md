# BaseMark 특허 우회 전략표

작성일: 2026-03-20  
목적: BaseMark가 출시 전후에 기능 구현을 계속하더라도, 특허 분쟁 가능성이 높은 영역을 미리 식별하고 우회 설계 원칙을 고정한다.

주의:
- 이 문서는 법률 자문이 아니다.
- 실제 침해 판단은 경쟁 특허의 청구항과 BaseMark 구현을 문장 단위로 비교해야 한다.
- 여기서는 제품 설계 단계에서 가장 실무적으로 유용한 1차 우회 기준만 정리한다.

## 1. 기본 원칙

BaseMark는 아래 방향을 기본값으로 유지한다.

- 자동 확정이 아니라 후보 생성 + 사람 확정
- 전역 3D 재구성이 아니라 국소 구간 비교
- 범용 장면 이해가 아니라 도면 기준 구조 비교
- 서버 중심이 아니라 로컬/온디바이스 중심
- 단일 사진 또는 짧은 현장 흐름 기준의 경량 비교

이 다섯 가지는 경쟁 제품과 유사해 보여도 BaseMark를 구조적으로 다르게 만드는 핵심 우회 축이다.

## 2. 위험 영역별 우회 전략

| 기능 영역 | 잠재 위험 | 분쟁이 붙기 쉬운 구현 | BaseMark 권장 우회 방향 |
| --- | --- | --- | --- |
| 사진 위 도면/기준 오버레이 | AR/오버레이 계열 특허 | 실시간 자동 정합, 자동 추적, 자동 고정 | 수동 기준점 2점 이상 지정 후 국소 정합, 사람 확인형 오버레이 |
| 거리/길이 측정 | AR 측정, 카메라 측정 특허 | 마커 없이 자동 절대 길이 산출, 실시간 전역 측정 | 기준 길이 입력 + 평면 계측 + 비율/환산 기반 mm 계산 |
| 도면-현장 자동 위치 연결 | 도면 매핑/자동 위치 추정 특허 | 완전 자동 위치 추정, 자동 현장 도면 링크 | 구간 선택 후 수동 앵커 정합, 필요시 보조 추천만 자동화 |
| 결함/차이 자동 판정 | 자동 검사 특허 | AI가 최종 불량 여부를 자동 확정 | `missing / extra / position_diff` 후보 생성까지만 자동화 |
| 실시간 카메라 추적 | SLAM/비주얼 포지셔닝 특허 | 지속 추적, 월드 락킹, 3D 앵커 고정 | 정지 프레임 또는 짧은 국소 프레임 기준 비교 |
| BIM/모델 중첩 | BIM/AR 시공 오버레이 특허 | 3D BIM 모델을 정확히 현장에 정합 | 2D 도면 기준 구조 앵커 + 국소 평면 정합 |
| 자동 객체 인식 | 특정 산업군 검사 특허 | 특정 설비를 자동 분류하고 판정까지 수행 | YOLO/SAM 등은 보조 후보 제안용으로만 사용 |
| 보고/기록 | 현장관리 SaaS 특허 | 협업 워크플로, 서버 동기화, 자동 할당 | 로컬 기록, 증적 저장, 사람 검토 중심으로 축소 |

## 3. BaseMark에 가장 맞는 안전한 구현 축

BaseMark는 아래 구현 축을 우선 채택한다.

1. 구조 앵커 선택
- 문틀 끝, 창호 바깥 프레임 모서리, 벽 코너처럼 잘 안 바뀌는 구조점을 사용한다.

2. 정규화 좌표
- 절대 픽셀이 아니라 `spanRatio`, `heightRatio`로 체크포인트를 정의한다.

3. 평면 계측
- 기준점 쌍과 기준 길이로 `px -> ratio -> mm` 환산을 수행한다.

4. 국소 정합
- 전체 공간 재구성이 아니라 현재 비교 구간만 맞춘다.

5. 후보 생성
- 결과는 `missing`, `extra`, `position_diff` 후보로 제한한다.

6. 사람 확정
- 최종 판정은 사람이 한다.

이 구조는 자동 검사, 자동 AR 측정, 자동 현장 이해 제품과 닮아 보이더라도 구현 책임이 다르다.

## 4. 우회가 특히 쉬운 지점

아래는 유사 특허가 있어도 BaseMark가 상대적으로 우회하기 쉬운 지점이다.

- 자동 정합을 수동 기준점 지정형으로 내릴 수 있다.
- 자동 절대 거리 추정을 기준 길이 입력형으로 바꿀 수 있다.
- 자동 판정을 후보 제안형으로 내릴 수 있다.
- 전역 공간 추적을 국소 평면 비교로 줄일 수 있다.
- 범용 인식을 구조 앵커 기반 비교로 좁힐 수 있다.

즉, BaseMark는 같은 문제를 풀더라도 청구항 핵심 요소를 바꿔서 빠져나갈 여지가 큰 편이다.

## 5. 반대로 조심해야 할 지점

아래 구현은 되도록 바로 코어에 넣지 않는다.

- 마커 없이 자동 절대 길이 측정
- 실시간 자동 정합 + 지속 추적
- 3D BIM 모델 실시간 중첩
- AI 자동 최종 판정
- 현장 위치 자동 인식 후 도면 자동 링크

이런 구현은 제품 데모는 강해 보일 수 있지만, 특허 리스크가 빠르게 커질 가능성이 있다.

## 6. 기능별 권장 우선순위

### 1순위
- 기준점 2개 이상 수동 지정
- 기준 길이 입력
- 비율/실길이 환산
- 후보 생성
- 사람 검토

### 2순위
- 기준점 자동 추천
- 관측 요소 자동 후보
- OCR 보조

### 3순위
- 실시간 카메라 보조
- AR 시각화
- 자동 위치 추정

## 7. 출시 전 1차 점검표

출시 전에는 아래 항목을 기능별로 점검한다.

1. 이 기능이 자동 정합/자동 추적/자동 판정인가
2. 이 기능이 절대 길이를 자동으로 산출하는가
3. 이 기능이 3D 모델 또는 전역 공간 정합을 전제로 하는가
4. 이 기능이 사람 확인 없이 결과를 확정하는가
5. 이 기능이 경쟁 서비스의 대표 동작과 너무 유사한가

하나라도 `예`가 많아지면 청구항 중심 검토 대상으로 올린다.

## 9. BaseMark 권장 결론

BaseMark는 아래 문장으로 방향을 고정한다.

> BaseMark는 자동 판정형 AR 검사 앱이 아니라, 도면 기준 구조 앵커와 정규화 좌표를 사용해 현장 장면의 국소 구간을 비교하고 후보를 생성한 뒤 사람이 최종 확정하는 시스템이다.

이 방향을 유지하면, 유사 제품이 있더라도 상당수 기능은 설계 단계에서 우회가 가능하다.
