# BaseMark 디자인 프롬프트

## 제품 개요

BaseMark는 현장 건설 기술자를 위한 안드로이드 앱이다.  
이 앱은 도면 기준 구조와 현장 사진을 비교해 차이 후보를 만들고, 사용자가 빠르게 기록하고 검토할 수 있게 돕는다.

## 사용자 프로필

- 현장 건설 기술자
- 감독자
- 품질 점검 담당자
- 모바일 앱을 길게 배우고 싶어하지 않는 사용자

## 금지 사항

아래 항목은 기본 화면에서 피한다.

1. 기술 용어를 한 화면에 많이 쌓지 말 것
2. 한 화면에서 여러 가지 작업을 동시에 요구하지 말 것
3. JSON, 디버그 값, 저수준 입력을 기본 화면에 두지 말 것
4. 탭과 버튼과 링크가 서로 중복된 흐름을 만들지 말 것
5. 터치가 어려운 작은 버튼을 두지 말 것
6. 사용자가 다음 단계가 무엇인지 추측하게 하지 말 것
7. 사진보다 설정 UI가 더 크게 보이게 하지 말 것
8. 설명 없이는 못 쓰는 화면을 만들지 말 것
9. 스크롤을 끝없이 요구하지 말 것
10. 자동 판정처럼 보이게 만들지 말 것

## 필수 요소

1. 모바일 우선
2. 한 번에 한 가지
3. 48px 이상 버튼
4. 현재 상태가 항상 보일 것
5. 사진 중심 화면
6. 오프라인 전제
7. 빠른 입력
8. 검토 결과가 명확할 것

## 명확한 흐름

### 1단계: 홈
- 현재 프로젝트
- 현재 상태
- 빠른 작업

### 2단계: 비교
- 사진 또는 카메라
- 기준 표시 찍기
- 보이는 대상 찍기

### 3단계: 후보 확인
- `missing`
- `extra`
- `position_diff`

### 4단계: 검토
- 확인
- 반려
- 보류

### 5단계: 저장/보고
- 로컬 저장
- 보고서

## 화면별 상세 설계

### 홈
- 상단: 프로젝트명, 상태 배지
- 중앙: 진행 요약
- 하단: 빠른 시작 버튼

### 비교
- 사진이 화면의 대부분을 차지
- 기준점은 사진 위에서 바로 찍고 드래그로 조정
- 첫 기준점을 찍으면 다음 기준점을 자동 안내
- 보이는 대상도 사진에서 바로 입력

### 후보 확인
- 후보는 카드형으로 나열
- 유형, 위치, 이유 코드, 검토 상태를 한 번에 읽을 수 있게

### 검토
- 확인/반려/보류
- 메모
- 검토 요약

### 관리/보고
- 저장된 시나리오
- 보고서
- 관리자용 고급 편집

## 구현 우선순위

### 1순위
- 홈 / 비교 / 검토 3화면
- 사진 중심 비교 화면
- 기준점/보이는 대상 입력
- 후보 검토

### 2순위
- 프로젝트 전환
- 보고서 화면
- 저장 시나리오 관리

### 3순위
- 관리자용 고급 편집
- 디버그 출력
- 저수준 계약 편집

## 검수 기준

아래 질문에 `예`가 많아야 한다.

1. 처음 보는 기술자가 도움 없이 다음 단계를 알 수 있는가
2. 사진이 화면의 중심인가
3. 버튼이 손가락으로 누르기 쉬운가
4. 현재 상태가 즉시 보이는가
5. 검토 화면이 별도로 분리되어 있는가
6. 기술 용어 없이도 기본 사용이 가능한가
7. 고급 입력은 뒤로 숨겨졌는가

## 구현 메모

현재 BaseMark 저장소에서는 아래 원칙으로 이 프롬프트를 적용한다.

- 엔진과 저장 로직은 유지
- 현재 웹 셸을 모바일 앱 셸처럼 재배치
- 고급 편집기는 관리자용으로만 유지
- 비교 코어는 사진 중심으로, 검토 코어는 카드 중심으로 단순화
