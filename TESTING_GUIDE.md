# BaseMark v1 UI - 실행 및 테스트 가이드

## 🚀 빠른 시작

### 1. 저장소 준비

```bash
# BaseMark 저장소 클론
git clone https://github.com/musin1045/basemark.git
cd basemark
git checkout master

# 의존성 설치
npm install
```

### 2. 새 UI 파일 통합

```bash
# 생성된 파일들을 저장소로 복사
# (이미 완료됨: index.html, styles.css, app.js)

# 확인
ls -lh src/ui/*.{html,css,js}
```

### 3. 개발 서버 시작

```bash
npm run app
```

출력:
```
Server running at http://localhost:3000
```

### 4. 브라우저에서 접속

```
http://localhost:3000
```

## ✅ 기능 테스트 (체크리스트)

### 1️⃣ 홈 화면 (Home Screen)

```
[ ] 홈 화면 표시됨
  ├─ [ ] 앱 제목 "BaseMark" 보임
  ├─ [ ] 앱 부제 "현장 검증 앱" 보임
  ├─ [ ] 영웅 이미지 보임
  └─ [ ] [검증 시작] 버튼 활성화

[ ] "어떻게 작동하나요?" 섹션
  ├─ [ ] 4단계 설명 표시됨
  └─ [ ] 텍스트가 명확함

[ ] "필요한 것" 섹션
  ├─ [ ] 도면, 사진, 시간 안내 표시됨
  └─ [ ] 텍스트가 명확함
```

### 2️⃣ Step 1: 기준 표시하기 (Verify Step 1)

```
[ ] 화면 진행률
  ├─ [ ] 진행바가 25% 표시
  └─ [ ] "단계 1 / 3" 텍스트

[ ] 캔버스 영역
  ├─ [ ] 캔버스가 표시됨
  ├─ [ ] 플레이스홀더 텍스트 보임
  └─ [ ] 크기가 적절함 (높이 60-70%)

[ ] 기준점 프리셋
  ├─ [ ] 6개 프리셋 버튼 표시됨
  │   ├─ [ ] 창호 좌상, 우상, 좌하, 우하
  │   └─ [ ] 벽 좌측, 우측
  ├─ [ ] 버튼 클릭 시 선택 표시
  └─ [ ] 지시문 업데이트

[ ] 사진 입력
  ├─ [ ] [📁 사진 업로드] 버튼
  │   └─ [ ] 클릭 시 파일 선택 대화 열림
  └─ [ ] [📷 카메라 켜기] 버튼
      └─ [ ] 클릭 시 경고 메시지 (준비 예정)

[ ] 기준점 표시
  ├─ [ ] 프리셋 선택 후 캔버스 클릭
  ├─ [ ] 기준점이 캔버스에 표시됨 (주황 점)
  ├─ [ ] 번호가 표시됨
  └─ [ ] 목록에 추가됨

[ ] "표시된 기준점" 목록
  ├─ [ ] 기준점 < 2개: 안내 텍스트
  ├─ [ ] 기준점 >= 1개: 목록 표시
  └─ [ ] [✕] 삭제 버튼 동작

[ ] 네비게이션 버튼
  ├─ [ ] [뒤로] 버튼 → 홈으로 돌아가기
  ├─ [ ] [다음] 버튼 (기준점 < 2 → 비활성화)
  └─ [ ] [다음] 버튼 (기준점 >= 2 → 활성화)
```

**테스트 절차:**
1. 파일 업로드 (sample.jpg 또는 사진)
2. "창호 좌상단" 프리셋 선택
3. 캔버스의 왼쪽 위 영역 클릭
4. "창호 우상단" 프리셋 선택
5. 캔버스의 오른쪽 위 영역 클릭
6. [다음] 버튼이 활성화되는지 확인

### 3️⃣ Step 2: 비교 실행 (Verify Step 2)

```
[ ] 화면 진행률
  ├─ [ ] 진행바가 66% 표시
  └─ [ ] "단계 2 / 3" 텍스트

[ ] 비교 진행 표시
  ├─ [ ] 스피너 애니메이션 표시
  └─ [ ] "비교 중..." 텍스트

[ ] 결과 요약 (비교 완료 후)
  ├─ [ ] 누락 카드 (📍)
  │   └─ [ ] 개수 표시 (예: "2개")
  ├─ [ ] 추가 카드 (➕)
  │   └─ [ ] 개수 표시 (예: "1개")
  └─ [ ] 위치 차이 카드 (📏)
      └─ [ ] 개수 표시 (예: "3개")

[ ] 네비게이션 버튼
  ├─ [ ] [뒤로] 버튼 → Step 1로 돌아가기
  └─ [ ] [결과 확인] 버튼 → Step 3로 이동
```

**테스트 절차:**
1. Step 1에서 기준점 2개 이상 표시
2. [다음] → Step 2로 이동
3. 스피너가 보이는지 확인 (약 1-2초)
4. 결과 요약이 표시되는지 확인

### 4️⃣ Step 3: 결과 확인 (Verify Step 3)

```
[ ] 화면 진행률
  ├─ [ ] 진행바가 100% 표시
  └─ [ ] "단계 3 / 3" 텍스트

[ ] 탭 필터링
  ├─ [ ] 4개 탭: 전체, 누락, 추가, 위치
  ├─ [ ] 탭 클릭 시 활성화 표시
  └─ [ ] 필터링된 결과 표시

[ ] 결과 목록
  ├─ [ ] 항목들이 리스트로 표시
  ├─ [ ] 각 항목에 유형, 제목, 상세정보
  └─ [ ] 항목 클릭 시 선택 표시

[ ] 항목 상세정보
  ├─ [ ] 선택된 항목의 상세 표시
  ├─ [ ] 유형, 항목 ID, 기준점, 이유 등
  └─ [ ] 정보가 정확함

[ ] 리뷰 액션 버튼
  ├─ [ ] [✓ 확인] 버튼 → 상태 저장
  ├─ [ ] [✕ 이상 있음] 버튼 → 상태 저장
  └─ [ ] [⏸ 보류] 버튼 → 상태 저장

[ ] 네비게이션 버튼
  ├─ [ ] [뒤로] 버튼 → Step 2로 돌아가기
  └─ [ ] [검증 완료] 버튼 → 완료 화면으로
```

**테스트 절차:**
1. Step 2에서 [결과 확인] → Step 3으로 이동
2. 결과 항목들이 표시되는지 확인
3. 탭 클릭으로 필터링 확인
4. 항목 클릭으로 상세정보 표시 확인
5. 리뷰 액션 버튼 클릭 시 메시지 표시 확인

### 5️⃣ 완료 화면 (Complete Screen)

```
[ ] 완료 아이콘
  ├─ [ ] 체크마크 아이콘 표시
  └─ [ ] 애니메이션 효과

[ ] 메시지
  ├─ [ ] "검증 완료!" 제목
  └─ [ ] "현장 검증이 저장되었습니다." 메시지

[ ] 통계
  ├─ [ ] 소요 시간 표시 (예: "2분 34초")
  └─ [ ] 발견된 항목 수 표시 (예: "8개")

[ ] 네비게이션 버튼
  ├─ [ ] [다시 검증하기] 버튼 → 홈으로
  └─ [ ] [모든 검증 보기] 버튼 → 경고 (준비 예정)
```

**테스트 절차:**
1. Step 3에서 [검증 완료] → 완료 화면으로 이동
2. 아이콘이 애니메이션으로 표시되는지 확인
3. 통계 정보가 정확한지 확인
4. [다시 검증하기] → 홈으로 돌아가기 확인

```bash
state

state.verification.candidates
```

### 네트워크 확인

```bash
# Network 탭에서 확인
- index.html (OK 200)
- styles.css (OK 200)
- app.js (OK 200)
- vendor/konva.min.js (OK 200)
- engine/ 파일들 (OK 200)
```

### 오류 해결

```
❌ Uncaught TypeError: generateComparisonCandidates is not a function
→ 엔진 파일 경로 확인

❌ Cannot find module './engine/baseMarkEngine.js'
→ app.js의 import 경로 확인

❌ Canvas not found
→ HTML에 id="canvas-anchor-setup" 확인
```
